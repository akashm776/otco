# Research Log

**Date:** 2026-04-22

---

# Question

Can OT-Mix outperform the SigLIP baseline on Flickr30K when fine-tuned from the baseline's best checkpoint?

---

# Current Understanding

Prior work (March–April) established that OT-Mix on Flickr8K with a partially-trained model showed OT negatives were semantically plausible but suffered from false-negative pressure (P(neg1 > GT) = 0.65). The hypothesis was that better encoder separation would reduce this pressure.

We now have a fully-trained Flickr30K baseline:
- Architecture: ResNet-50 + DistilBERT → 512-d L2-normalized, temp=0.07
- Loss: SigLIPLoss (sigmoid contrastive over all B² pairs with learned logit_bias)
- Best epoch 48/50, Canonical Avg R@1 = **32.10%**
- SigLIP geometry: raw cosine positive pairs ≈ +0.016, negatives ≈ −0.40 (logit_bias dominates)

The plan: fine-tune from `best_model.pt` with `SoftmaxMixLoss` (OT-Mix) for 30 epochs.

---

# Gap

The OT cost matrix in `SoftmaxMixLoss` was `C_ij = 1 - cos_sim`, calibrated for cosine space with `ot_eps=0.05`. With SigLIP, cosine similarities cluster tightly near 0 (range ≈ 0.27). This is a much smaller range than the 0–2 range assumed when setting `ot_eps=0.05`.

---

# Hypothesis

Cosine-space cost matrix range ≈ 0.27 → Gibbs kernel `exp(-C/ot_eps)` with `ot_eps=0.05` is nearly uniform over that range → Sinkhorn produces near-uniform plans → barycentric negatives ≈ mean of all negatives → useless signal.

---

# Executable Test

Run `softmax_mix_k32_from_baseline` (cosine-space OT, ot_eps=0.05) for 1–2 epochs and inspect OT-MIX STATE metrics from training log.

Then fix: If as feared then move cost matrix to logit space. SigLIP logit = `cosine_sim * scale + logit_bias` where scale ≈ 14.3. Logit-space cost: `C_ij = max_logit - logit_ij`, range ≈ 2.75 (10× cosine). Recalibrate `ot_eps = 0.05 × 14 ≈ 0.7`. Gate must also move to logit space: `gate_sim = -4.0` (avg negative logit ≈ −5.75, so −4.0 passes genuinely hard synthetics).

---

# Result

Cosine-space run (softmax_mix_k32_from_baseline), epoch 1:
- coupling_entropy ≈ 3.33 (≈ log(28), near-uniform — effective k < 32 due to masking)
- peak_mass ≈ 0.062 (≈ 1/16, near-uniform)
- selected_rank ≈ 17 (uniform would select rank 16)
- Epoch 1 Canonical Avg R@1: **26.50%** (degraded from 32.10% baseline start)

Confirms uniform plan hypothesis. Cosine-space OT is completely degenerate with SigLIP embeddings.

Implemented logit-space OT (`softmax_mix_k32_logit_from_baseline`, ot_eps=0.7, gate_sim=−4.0):
- coupling_entropy ≈ 2.0–2.5 (sharp, non-uniform)
- peak_mass ≈ 0.40–0.43 (one negative dominates per row)
- selected_rank ≈ 1.4–1.6 (selecting genuinely hardest negatives)
- avg_synthetic_logit ≈ −3.8 to −4.3 (harder than avg negative at −5.75)

---

# Structural Update

**Key insight:** SigLIP's `logit_bias` compresses the entire discrimination signal into a single learned scalar. Cosine similarities cluster near zero regardless of semantic content. The OT cost matrix must be built from logits (not cosines) to have any discriminative range.

**Code changes made (`model/loss.py`):**
- `_make_plan` now takes `logits_biased` as input; cost = `(max_logit - logits_biased).clamp_min(0)`
- Gate moved to logit space: `gate = (synth_logits > self.gate_sim).float()`
- New loss_dict keys: `avg_synthetic_logit`, `ot_step`
- `ot_eps=0.7` (logit-space calibration), `gate_sim=-4.0`

**Configs added:**
- `softmax_mix_k32_logit_from_baseline` (α=0.1)
- `softmax_mix_k32_from_baseline` kept for reference (cosine-space, degenerate)
