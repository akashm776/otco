# Question

Does a larger random pool (N=256) improve retrieval over N=128, or does the random sampling strategy introduce proportionally more irrelevant negatives as N grows?

The live re-forward experiment confirmed that the directional asymmetry in v1 (T→I +0.31pp, I→T −0.10pp vs baseline) is consistent with gradient routing through the text encoder only — adding image-side gradient restores I→T but removes the T→I advantage, leaving net Avg unchanged.

---

# Gap

N=128 pool sampled from 5994 training images: with 200 classes (~30 images/class), each step draws ~0.64 images per class on average. Many classes will be absent from any given N=128 sample. Doubling to N=256 gives ~1.28 images/class per step, increasing the probability of near-species candidates being present for OT to select.

The risk: a larger random pool also includes proportionally more distant-class images. OT plan entropy may increase (more diffuse allocation) → adaptive gate suppresses more steps → effective α drops.

---

# Hypothesis

N=256 improves pool coverage and negative quality:
- `pool_selected_rank_mean` will be slightly higher in absolute terms (more candidates, absolute rank scales) but lower as a fraction of pool size
- `coupling_entropy` may increase slightly (more candidate diversity → more diffuse allocation)
- If diffuse suppression is manageable: I→T should improve (harder negatives selected), T→I direction may shift slightly given the same detached asymmetry

Expected outcome: Avg R@1 ~1.49–1.52%, possibly improving the current best.

Falsifiable: if `gap_bucket_diffuse` grows to dominate (useful < diffuse in ep30+), then the larger pool is hurting plan quality more than it is helping selection quality.
