# Question

Does cached-pool OT-Mix (N=128, detached) beat adaptive gated OT-Mix? And if it does, what explains the directional asymmetry — and can restoring image-encoder gradient through live re-forward of OT contributors fix it?

---

# Current Understanding

Cached-pool OT-Mix N=128 (`cub200_softmax_mix_cached_pool_128`) completed — 50 epochs:

| Epoch | T→I R@1 | I→T R@1 | Avg R@1 |
|---|---|---|---|
| 10 | 0.54% | 0.59% | 0.56% |
| 20 | 0.72% | 0.86% | 0.79% |
| 30 | 0.85% | 1.16% | 1.00% |
| 43 | 1.21% | 1.50% | 1.36% |
| **49** | **1.36%** | **1.61%** | **1.48%** |
| 50 | 0.71% | 1.59% | 1.15% |

Best = epoch 49, **1.48%** — new best, +0.10pp over adaptive gated (1.44%), +0.10pp over baseline (1.38%).

OT diagnostic summary (mature training, ep30–50):
- `pool_selected_rank_mean`: 3–7 (genuinely hard, vs batch-local ~31/64)
- `coupling_entropy`: 2.87–3.10, useful bucket dominant in 40/41 logged steps
- Gap bucket distribution: useful=63, diffuse=21, too_easy=4, too_hard=2

The cached pool meaningfully improves OT negative quality. Selected rank 3–7 vs ~31 confirms the pool is finding near-species negatives that batch-local sampling misses.

**Directional asymmetry:**
- T→I gain: +0.31pp over baseline (1.36% vs 1.05%) — strong
- I→T: −0.10pp vs baseline (1.61% vs 1.71%) — slight degradation

Interpretation: the synthetic loss uses detached pool embeddings → gradient flows only through the text encoder and not through the image encoder in the synthetic branch. Base SigLIP provides image encoder gradient, but the synthetic branch trains text exclusively. This is consistent with T→I improving (text queries become better) while I→T declines (image representations are not additionally shaped by OT).

---

# Gap

The detached pool was a deliberate design choice (avoid computing gradients through 5994 cached embeddings). But the asymmetry shows a real cost: image representations are not updated by the OT objective.

Two candidate fixes:
1. **Live re-forward of top OT contributors**: use the cached pool for OT plan computation (no_grad), but then select the top-mass contributors per query, re-forward only those images through the live encoder, and build the barycentric synthetic from live features. Gradient flows to the image encoder.
2. **Larger detached pool**: more candidates → more hard negatives for the text encoder → stronger T→I gain, but asymmetry persists.

---

# Hypothesis

Live re-forward (Phase III v2) will restore I→T toward the baseline level (1.71%) while preserving or improving T→I, because both encoders now receive OT-shaped gradient. Expected outcome: T→I ~1.30–1.40%, I→T 1.70%+, Avg ~1.50%+.

Falsifiable: if I→T recovers to ~1.71% but T→I drops relative to v1, then the gradient asymmetry was indeed the mechanism behind T→I's gain in v1 (detached pool trains text harder, which benefits T→I).

# Executable Test

**New experiment:** `cub200_softmax_mix_live_reforward_pool_128`

Same config as `cub200_softmax_mix_cached_pool_128` with `live_reforward: true`, `live_top_m: 2`, `max_live_reforward: 64`.

Implementation:
1. Per step: compute OT plan over pool (no_grad, detached)
2. `_select_live_contributors()`: select top-2 contributors per query by OT mass → deduplicate → cap at 64 by aggregate mass → map back to dataset indices
3. Re-forward selected images through live `model.encode_images()` → normalize → `live_feats` has gradient
4. Build barycentric synthetic from `live_feats` (gradient-enabled) instead of cached pool embeddings
5. Log `num_live` (unique images re-forwarded per step) and `mass_retained` (fraction of OT plan mass captured by live set)

Cost: ≤64 extra forward passes per step (same as one batch), total ≤128 images/step.

| Epoch | T→I R@1 | I→T R@1 | Avg R@1 |
|---|---|---|---|
| 10 | 0.38% | 0.66% | 0.52% |
| 20 | 0.59% | 0.95% | 0.77% |

Best = epoch 40, **1.44%**.

Key diagnostic: `mass_retained` ≈ 0.50 throughout. The cap at `max_live_reforward=64` is binding — only ~50% of OT plan mass is captured by the live contributor set. The remaining mass is in long-tail contributors that are not re-forwarded.

# Structural Update

Phase III v2 confirms the asymmetry hypothesis. Live re-forward restores I→T to **1.71%** (matching baseline exactly) but loses T→I gain: **1.17%** vs v1's **1.36%**. Net Avg R@1: 1.44% — matching adaptive gated, not beating detached v1 (1.48%).

**Interpretation:** Detached cached-pool OT acts primarily as text-side alignment pressure. Adding image-side gradient through live re-forward makes the objective more symmetric, but symmetry does not improve Avg retrieval in this setting. The observed I→T shift is consistent with the gradient-routing change, but the claim is observational — the diagnostics support this interpretation, not confirm it mechanistically.

**Updated comparison table:**

| Setting | T→I R@1 | I→T R@1 | Avg R@1 |
|---|---|---|---|
| Baseline | 1.05% | 1.71% | 1.38% |
| Adaptive gated | 1.19% | 1.69% | 1.44% |
| Cached-pool N=128 detached (v1) | **1.36%** | 1.61% | **1.48%** |
| Live re-forward N=128 (v2) | 1.17% | **1.71%** | 1.44% |

Detached cached-pool v1 remains the best overall result.
